### 3）目标线路 SUMO 场景建模（先能对齐“站点”）
- [ ] 明确研究对象
  - [ ] 选定 1 条公交线路（含方向）与 1 个代表性时段（如早高峰 7:30–9:00）
  - [ ] 列出站点序列、站间距离、首末站信息（来自高德/开放数据）
- [ ] 路网准备（任选一条可执行路径）
  - [ ] Path A：OSM/公开路网 → `netconvert` 生成 `*.net.xml`
  - [ ] Path B：已有路网文件直接导入（若你已有现成 `*.net.xml`）
- [ ] 公交站点与线路文件
  - [ ] 在 `sumo/additional/` 生成公交站点（`busStop`）并绑定到 lane
  - [ ] 在 `sumo/routes/` 建立线路 route 与车辆 flows（先用少量车辆验证）
- [ ] 运行基线仿真并产出“站点层”日志
  - [ ] 至少能导出：到站/离站时刻、停站时间、站间运行时间
  - [ ] 保存：`sumo/config/*.sumocfg`、仿真输出（tripinfo/stopinfo 等）

### 4）高德真值数据提取（站点时刻 + 路况速度）
- [ ] 明确 API 需求与字段映射（在 `docs/gaode_schema.md` 记录）
  - [ ] 线路/站点：线路 id、站点序列、站点坐标、首末班
  - [ ] 到离站/ETA：能否获得“站点时刻/到站预测”（若只能取 ETA，需定义真值替代方案）
  - [ ] 路况：路段平均速度/拥堵等级、时间戳、空间匹配方式
- [ ] 编写“下载器”脚本（只做抓取与落盘，不做复杂逻辑）
  - [ ] 原始响应落 `data/raw/`（按 `线路-方向-日期-时段.json` 命名）
  - [ ] 日志记录：请求参数、返回码、限流与重试策略
- [ ] 编写“清洗器”脚本（输出结构化真值）
  - [ ] 站点层：生成站点到离站时间序列（统一时区/格式、去重、补缺）
  - [ ] 路段层：生成速度时间序列（异常值剔除、重采样、平滑策略可选）
  - [ ] 输出到 `data/processed/`（推荐 CSV/Parquet）
